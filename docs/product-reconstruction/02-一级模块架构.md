# 客户端一级功能架构

## 1. 一级功能结构

| 一级模块 | 产品定位 | 主要用户 | 核心价值 | 复杂度等级 | 当前完整度 | 是否建议重构 |
|---|---|---|---|---|---|---|
| 客户端壳层与全局导航 | 承载主框架、导航、窗口、全局弹窗和系统托盘 | 全部用户 | 提供客户端统一入口和窗口生命周期 | L4 | 高 | 建议抽象为全局框架域 |
| 登录与账号体系 | 控制进入客户端、自动登录、登录过期、退出、安全提醒 | 全部用户 | 建立所有业务访问前提 | L4 | 高 | 建议与初始化解耦 |
| 登录后初始化与资源状态 | 加载用户、区域、资源、公告、版本和白名单 | 全部用户 | 决定模块可见性和可用性 | L5 | 高 | 建议沉淀全局状态中心 |
| SClaw / 智能助手 | 提供 AI 对话、技能、Agent、频道、定时任务、模型配置 | 白名单用户 | 提供智能辅助和日志分析能力 | L5 | 中高 | 建议保持独立一级域 |
| 应用与软件服务 | 管理应用中心、我的应用、作业提交、图形提交 | 计算用户 | 从软件入口启动计算任务 | L5 | 高 | 保持核心入口 |
| 作业管理 | 管理当前、历史、提交中作业和作业详情 | 高频计算用户 | 支撑计算任务全生命周期 | L5 | 高 | 保持核心业务域 |
| 命令行 / Shell | 提供终端、多标签、分屏、文件侧栏和软件环境 | 高级用户 | 支持命令式集群操作 | L4 | 高 | 保持独立窗口形态 |
| 图形模块 | 管理 VNC、DCV、RDP、VM 和图形作业 | GUI 软件用户 | 支持交互式图形计算 | L5 | 高 | 统一会话模型 |
| 文件模块 | 管理远程文件、上传下载、跨域、压缩、分享、预览 | 全部计算用户 | 支撑输入输出数据流转 | L5 | 高 | 保持核心业务域 |
| 个人中心与账号安全 | 展示个人资料、安全等级、实名、手机、邮箱、微信状态 | 全部用户 | 提升账号可信度和安全性 | L3 | 中高 | 可归入用户域 |
| 费用中心 / 我的资源 / 资源申请 | 提供费用、资源、购买、申请和无资源引导 | 管理员、缺资源用户 | 解决资源不足导致的阻断 | L4 | 中高 | 形成统一资源引导域 |
| 帮助、反馈、客服、日志 | 提供文档、反馈、咨询、日志和排障入口 | 全部用户、客服支持 | 降低使用和排障成本 | L3 | 高 | 合并为支持服务域 |
| 更新机制 | 检查、提示、下载和安装客户端更新 | 全部用户 | 保证版本安全和维护性 | L4 | 高 | 保持全局能力 |
| 全局任务中心 | 聚合上传、下载、跨域、压缩和异步文件任务 | 文件与数据用户 | 让长耗时任务可见可控 | L5 | 高 | 建议独立一级业务域 |
| 权限、角色、区域维护、资源状态 | 控制入口、按钮、空态、阻断和错误提示 | 全部用户 | 保证用户进入可用路径 | L5 | 中高 | 统一规则字典 |
| 数据埋点 | 记录登录、SClaw、应用、作业、文件、Shell 等事件 | 产品、运营、研发 | 支持产品分析和质量监控 | L3 | 中高 | 建立事件字典 |
| 其他隐藏功能 | 公告、引导、工具下载、托盘、异常页、免责声明 | 全部用户、内部支持 | 补齐横向辅助体验和风险控制 | L4 | 中 | 建立隐藏功能台账 |

## 2. 模块之间的关系

| 关系 | 描述 |
|---|---|
| 登录 → 初始化 | 登录成功后加载用户、角色、区域、资源、公告、版本和 SClaw 白名单 |
| 初始化 → 壳层 | 初始化结果决定左侧导航、顶部入口、用户菜单和全局弹窗 |
| 权限 → 全模块 | 角色、区域、资源账号和维护状态影响应用、作业、Shell、图形、文件、任务中心 |
| 应用 → 作业 | 应用和模板提交会生成作业，并跳转作业管理 |
| 应用 → 图形 | 图形类应用可创建 VNC/DCV/RDP/VM 或 GUI 作业 |
| 作业 → 文件 | 作业详情和输出可进入工作目录、结果文件、下载和压缩 |
| 作业 → SClaw | 作业日志和输出支持 AI 分析辅助 |
| 文件 → 任务中心 | 上传、下载、跨域、压缩和异步文件任务进入任务中心 |
| 文件 → Shell | 文件右键可打开 Shell 并进入目录，Shell 内嵌文件侧栏 |
| 文件 → 图形 | 文件右键存在 VNC/DCV 后处理入口 |
| Shell → 文件 | Shell 支持文件侧栏、拖拽路径、rz 上传和下载任务 |
| 图形 → 独立窗口 | VNC/DCV/RDP/VM 通过独立窗口承载远程桌面 |
| 资源 → 应用/作业/图形/文件/Shell | 无资源、区域维护、账号停用会阻断核心流程 |
| 帮助/日志 → 全模块 | 用户遇到异常时可打开帮助、反馈、客服和日志 |
| 更新 → 壳层 | 强更新阻断全局使用，弱更新在顶部和关于弹窗提示 |
| 埋点 → 全模块 | 登录、SClaw、应用、作业、文件、Shell、引导事件进入统计 |

## 3. 模块重构分层建议

| 层级 | 建议归属 | 包含模块 |
|---|---|---|
| 全局基础层 | 壳层、登录、初始化、权限、更新、公告、托盘、异常页 | 01、02、03、13、15、17 |
| 核心业务层 | 应用、作业、文件、Shell、图形、任务中心 | 05、06、07、08、09、14 |
| 智能辅助层 | SClaw、AI 日志分析、智能推荐、AI 排障 | 04 及各模块 AI 入口 |
| 账号资源层 | 个人中心、费用、我的资源、资源申请、实名和安全 | 10、11 |
| 支持运营层 | 帮助、反馈、客服、日志、埋点 | 12、16 |

## 4. 架构风险点

| 风险 | 涉及模块 | 影响 |
|---|---|---|
| 资源状态分散 | 初始化、权限、应用、作业、文件、图形、Shell | 重构时容易出现入口错显和操作误放行 |
| 任务模型分散 | 文件、任务中心、壳层 | 上传、下载、跨域、压缩的状态和数量口径可能不一致 |
| 图形会话类型多 | 图形、应用、作业、壳层 | VNC/DCV/RDP/VM 差异影响会话统一体验 |
| SClaw 宿主和子应用边界复杂 | SClaw、壳层、埋点、作业 | 白名单、服务状态、消息桥接和日志分析需明确边界 |
| 控制台外链承载较多 | 个人中心、资源、费用、反馈 | 客户端内外边界需在 PRD 中明确 |
| 隐藏能力容易遗漏 | 公告、工具下载、托盘、开发者面板、免责声明 | 迁移时可能损失排障和合规能力 |
