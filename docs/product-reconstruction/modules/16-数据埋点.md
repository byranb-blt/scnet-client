## 1. 模块定位

数据埋点模块用于记录客户端用户行为、登录转化、SClaw 使用、应用/作业/图形/文件/Shell 操作、页面访问和调试信息。它面向产品分析、增长漏斗、问题定位和体验优化，用户侧通常无明显界面，开发者侧可通过面板打开神策调试日志或查看 SClaw 本地埋点。

## 9. 模块联动关系

- 数据埋点覆盖登录、SClaw、应用、作业、图形、文件、Shell、新手引导等主要模块。
- 登录模块提供匿名 ID、登录结果和用户识别，影响后续事件归因。
- SClaw 通过宿主桥接把子应用事件转成客户端神策事件。
- 子模块通过全局 `$sensorsTrack` 复用上报能力。
- 开发者面板可打开神策调试日志，SClaw 自身保留本地 UX/性能埋点查看器。

## 12. 面向重构的产品建议

| 建议 | 说明 |
|---|---|
| 保留事件命名稳定性 | 重构时应建立事件映射表，避免历史数据断层 |
| 明确客户端统计口径 | Web、客户端、子模块、SClaw 的上报边界需产品确认 |
| 增加埋点字典文档 | 每个事件应记录触发点、字段、业务含义和所属模块 |
| 保留登录漏斗 | 登录结果和失败原因对客户端质量监控重要 |
| 强化合规说明 | 行为采集和第三方统计需要在隐私政策或设置中有明确说明 |

## 本模块产品还原结论

- 已确认功能：神策初始化、公共属性、用户画像、匿名 ID、登录漏斗、SClaw 事件、应用/作业/图形事件、文件事件、Shell 事件、百度统计、开发者调试日志。
- 新增发现功能：SClaw 子应用通过宿主转发神策事件、SClaw 本地 UX/性能埋点查看、非生产环境神策日志开关。
- 待确认功能：客户端与 Web 的统计口径、隐私合规提示入口、子模块 `$sensorsTrack` 在客户端侧的实际生效范围。
- 疑似废弃功能：无明确疑似废弃功能。
- 复杂度最高的功能：跨模块统一埋点和 SClaw 子应用事件桥接。
- 最适合 AI 接管或辅助的功能：埋点字典生成、事件字段解释、数据异常排查、漏斗问题定位。
- 重构时最需要保留的能力：登录结果、核心业务操作、SClaw 使用、文件上传结果、Shell 关键操作的连续上报。
- 重构时可以弱化或合并的能力：开发者调试日志可统一归入开发者面板。
