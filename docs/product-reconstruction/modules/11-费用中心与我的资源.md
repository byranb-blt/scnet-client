## 1. 模块定位

费用中心、我的资源和资源申请模块为客户端提供资源和费用管理入口，并在用户缺少计算资源、资源与应用不匹配、配置订单处理中、资源申请待审批等场景下给出引导。客户端内主要承载入口、阻断提示和资源开通引导，费用账单、基础资源管理、资源购买与审批流程主要由 Web 控制台承载。

## 6. 状态与规则

| 状态对象 | 状态值 | 用户可见表现 | 可执行操作 | 不可执行操作 | 备注 |
|---|---|---|---|---|---|
| 资源状态 | 有可用资源 | 可进入文件、作业、应用提交 | 使用资源相关功能 | 无 | 正常状态 |
| 资源状态 | 无区域资源 | 弹出开通计算资源 | 购买资源、查看订单、刷新 | 文件/作业主流程受阻 | 全局阻断 |
| 订单状态 | 配置中 | 提示预计 1-5 分钟 | 查看订单、刷新 | 立即使用资源 | 资源配置未完成 |
| 申请状态 | 未处理 `NoHandle` | 提示已提交申请 | 等待、刷新 | 立即使用资源 | 审批中 |
| 申请状态 | 处理中 `Handling` | 提示申请处理中 | 等待、刷新 | 立即使用资源 | 审批或配置中 |
| 申请状态 | 等待自动审批 `WaitAutoApprove` | 提示等待审批 | 等待、刷新 | 立即使用资源 | 自动流程状态 |
| 申请状态 | 失败 `Failed` | 提示申请失败或需要重新处理 | 重新申请或联系支持 | 继续使用该申请 | 失败分支 |
| 用户角色 | 团队管理员 | 可购买资源、查看管理员费用 | 购买、查看订单 | 无 | 购买能力主体 |
| 用户角色 | 普通成员 | 提示联系团队管理员 | 查看提示 | 购买资源入口受限 | 团队资源由管理员管理 |
| 用户角色 | BGP 操作员 | 费用中心隐藏 | 进入允许模块 | 查看费用中心 | 特定角色限制 |
| 应用资源匹配 | 资源支持应用 | 可选择资源提交 | 提交作业 | 无 | 应用提交正常 |
| 应用资源匹配 | 已有资源不支持应用 | 展示支持区域和申请提示 | 管理员按区域购买 | 直接提交 | 应用提交阻断 |

## 9. 模块联动关系

- 资源状态决定文件模块、当前作业、应用提交、Shell、图形等核心功能是否可用。
- 费用中心和我的资源入口与个人中心同处顶部用户菜单，受角色与 BGP 规则影响。
- 应用提交页通过资源选择表和应用内无资源提示连接资源申请流程。
- 无资源弹窗与订单、申请单、团队角色联动，决定展示购买、继续购买、查看订单或联系管理员。
- 资源购买、申请、审批、配置变更主要跳转 Web 控制台承载，客户端保留入口和状态反馈。

## 12. 面向重构的产品建议

| 建议 | 说明 |
|---|---|
| 将资源引导做成统一能力 | 文件、作业、应用、图形等模块应复用同一资源状态与引导模型 |
| 明确角色差异文案 | 团队管理员和成员的购买/联系管理员路径应统一表达 |
| 拆分客户端内置与控制台承载 | 客户端保留入口与阻断提示，控制台承载购买、审批、订单、配置变更 |

| 保留刷新资源能力 | 资源配置和订单完成后，用户需要在客户端内刷新状态继续流程 |
