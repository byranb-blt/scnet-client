## 1. 模块定位

文件模块是客户端的计算数据管理中心，负责跨区域文件浏览、上传下载、文件操作、分享、跨域传输、压缩解压、预览、任务跟踪，并为 Shell、应用、作业、图形提供文件选择和路径联动能力。

核心用户场景包括：进入资源区域查看家目录/共享目录、上传输入文件、下载计算结果、复制/移动/删除/重命名文件、分享目录或文件、跨区域传输数据、压缩或解压大量文件、从文件路径打开 Shell、从文件选择器为应用或作业选择输入输出路径。

## 9. 模块联动关系

- 文件模块与资源模块联动：区域、账号状态、存储配额、多区域资源决定文件浏览、上传下载、跨域、压缩解压可用性。
- 文件模块与 Shell 模块联动：右键可从当前目录打开 Shell，Shell 侧边栏可浏览文件，上传完成会刷新 Shell 文件树。
- 文件模块与应用模块联动：应用提交和软件发布场景复用文件选择器，用户从文件系统选择输入文件、脚本、结果目录。
- 文件模块与作业模块联动：压缩/解压通过作业方式执行，作业结果和输出文件回到文件模块查看与下载。
- 文件模块与图形模块联动：右键菜单存在图形后处理入口，可基于文件打开 VNC/DCV 类处理流程。
- 文件模块与全局任务中心联动：上传、下载、跨域、压缩解压任务在顶部入口和独立传输窗口中展示。
- 文件模块与客户端壳层联动：系统文件选择、下载目录、打开本地文件夹、传输窗口关闭提醒依赖 Electron 客户端能力。

| 传输窗口上传 Tab 打开事件 | 待确认 | `src/views/transfer/index.vue` 存在表达式式写法，需确认真实交互 |

## 12. 面向重构的产品建议

| 建议 | 说明 |
|---|---|
| 统一任务模型 | 上传、下载、跨域、压缩解压、异步复制移动删除应统一 `类型/状态/进度/来源/目标/错误/可操作项` |
| 固化右键菜单规则 | 右键菜单由角色、权限、文件类型、选择数量、路径决定，建议形成可配置规则表 |
| 提升跨域传输可解释性 | 跨域涉及源区域、目标区域、账号状态、同名策略和失败重传，需要清晰展示原因和下一步 |
| 将文件选择器作为平台能力管理 | 应用、作业、Shell 都依赖文件选择，应定义统一选择范围、权限和返回数据 |
| 保留客户端本地能力 | 系统文件选择、下载目录、打开本地文件夹、关闭提醒是客户端体验核心 |
| 强化存储容量和账号异常去向 | 存储不足、账号禁用、区域维护应提供资源中心或联系入口 |
| 用 AI 辅助异常处理 | 上传失败、跨域失败、压缩失败、权限不足、存储不足都适合由 AI 解释原因并推荐路径 |

## 本模块产品还原结论

- 已确认功能：文件浏览、路径导航、区域切换、基础文件操作、上传下载、快速传输、跨域传输、压缩解压、文件分享、文件预览、右键菜单、Shell 联动、文件选择器复用、顶部任务入口。
- 新增发现功能：Linux 快传命令、异步复制/移动/删除任务记录、图形后处理入口、BGP 角色隐藏文件能力、文件新手引导、上传完成刷新跨域源/目标列表。
- 待确认功能：顶部任务入口对跨域和压缩任务数量的覆盖范围、图形后处理线上适用范围、传输窗口上传 Tab 打开事件、压缩取消路径、异步任务保留周期。
- 疑似废弃功能：旧上传/下载独立窗口与统一传输窗口并存，部分客户端上传下载注释链路需要确认保留边界。
- 复杂度最高的功能：跨域传输、分片上传、压缩/解压作业、权限驱动右键菜单。
- 最适合 AI 接管或辅助的功能：上传失败诊断、跨域失败解释、存储不足处理建议、权限不足解释、压缩/解压失败分析。
- 重构时最需要保留的能力：跨区域文件浏览、稳定上传下载、任务进度追踪、权限保护、Shell/应用/作业文件选择联动。
- 重构时可以弱化或合并的能力：分散的上传/下载/传输窗口入口、重复的任务列表呈现、低频图形后处理入口、旧版独立上传下载窗口。
