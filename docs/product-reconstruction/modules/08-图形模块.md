## 1. 模块定位

图形模块为用户提供远程桌面、图形软件运行、图形作业提交和虚拟机访问能力。它承接“应用模块”的图形类软件入口，也独立提供当前会话、历史会话和图形应用中心入口。该模块的核心价值是让用户在客户端内启动、进入、管理和回收 GUI/VNC/DCV/RDP/VM 类型的交互式计算环境。

主要场景包括：从图形应用中心启动软件、从应用详情提交图形作业、打开已创建的远程桌面、管理排队或运行中的图形会话、停止/删除/续期图形实例、查看历史图形会话。

## 9. 模块联动关系

- 图形模块与应用模块共享图形应用入口，应用交付方式决定创建 VNC、DCV、RDP、VM 或模板会话。
- 图形作业提交会创建 GUI 作业，并与作业模块的排队、运行、取消、日志能力联动。
- 图形模块依赖资源模块提供区域、集群、账号、队列、预算、存储、会话数量限制等前置条件。
- 图形远程桌面依赖客户端壳层打开独立窗口，并通过 `/vnc`、`/vm` 承载远程访问页面。
- VM 停止或删除会通知已打开的 VM 窗口关闭，形成会话管理与窗口生命周期联动。
- 图形会话卡片中的日志能力连接作业日志或图形会话日志，后续可与 SClaw 日志分析形成辅助路径。
- 区域维护、账号禁用、软件未安装等全局状态会影响图形入口可见性与创建按钮可用性。

## 12. 面向重构的产品建议

| 建议 | 说明 |
|---|---|
| 建立统一图形会话模型 | 将 VNC/DCV/RDP/VM 的会话状态、操作按钮、可打开条件统一表达，减少用户理解成本 |
| 拆清“图形作业”和“图形会话” | 用户需要知道排队的是作业，打开的是会话，二者关系应在界面上明确 |
| 保留独立窗口生命周期联动 | 远程桌面窗口打开、VM 停止/删除关闭窗口，是客户端体验的关键能力 |
| 强化资源阻断提示 | 区域维护、账号禁用、会话数量超限、存储不足应提供统一的去向和解决建议 |
| 梳理旧新会话接口边界 | 新 GV、旧 GV、desktop center、desktopagent 的入口边界需要形成产品规则 |
| 将图形入口聚合到应用中心 | 应用模块和图形模块都存在启动入口，重构时应建立统一的入口策略 |
| 适配 AI 辅助 | 图形创建失败、排队原因、资源不足、远程桌面连接失败适合接入智能解释与下一步建议 |

## 本模块产品还原结论

- 已确认功能：图形会话列表、历史会话、图形应用中心、VNC/DCV/RDP/VM 创建与打开、图形作业提交、会话停止/删除/续期/日志/截图、独立远程桌面窗口。
- 新增发现功能：DCV 访问文件下载、VM 远程连接就绪检查、VM 停止/删除联动窗口关闭、Frame/VNC 模板创建、图形提交后自动等待运行并匹配会话。
- 待确认功能：ICA/Frame 线上入口、当前列表表格视图、RDP 多版本策略、新旧图形会话删除边界、GUI 作业运行后无会话匹配的标准文案。
- 疑似废弃功能：ICA 改造相关分支和部分列表视图切换痕迹需要结合线上配置确认。
- 复杂度最高的功能：VNC/DCV 独占会话创建与 GUI 作业运行后自动打开远程桌面的链路。
- 最适合 AI 接管或辅助的功能：图形创建失败解释、资源不足处理建议、排队原因说明、远程桌面连接异常诊断。
- 重构时最需要保留的能力：多类型图形会话创建、状态驱动操作按钮、独立窗口打开、作业与会话匹配、资源阻断提示。
- 重构时可以弱化或合并的能力：旧版会话删除差异、重复的图形入口、历史列表中低频筛选项、Frame/ICA 历史分支。
