# 模块名称：更新机制

## 0. 本模块检查过的代码位置

| 代码位置 | 识别内容 |
|---|---|

| `src/Layout/index.vue` | 主框架挂载、顶部弱更新提示、2 小时轮询检查 |
| `src/Layout/components/aboutDialog.vue` | 关于弹窗、版本信息、升级按钮、红点 |

| `src/api/login.js` | 服务端版本检查接口 |

| `main/dev-panel/index.js` | 开发者更新配置 IPC |

## 1. 模块定位

更新机制用于保证客户端版本及时升级，并在强更新场景阻断继续使用，在弱更新场景提供顶部提示、关于弹窗提醒和手动升级入口。该模块同时包含下载完成后的安装升级动作，以及开发者调试升级链路的 Feed、缓存目录和版本覆盖能力。

## 2. 识别出的页面、入口、组件、弹窗

| 类型 | 名称 | 位置/证据 | 承载功能 |
|---|---|---|---|
| 顶部提示 | 弱更新提示条 | `Layout/index.vue` | 发现新版本、显示版本号、可关闭 |
| 用户菜单提示 | 关于红点 | `SubMenu.vue` | 提醒关于页存在新版本 |
| 弹窗 | 关于 | `aboutDialog.vue` | 当前版本、最新版本、发布日期、更新说明、立即升级 |

| 系统能力 | 自动下载与安装 | `main/updater/index.js` | 检查更新、下载、下载完成、安装退出 |
| 开发者面板 | 更新调试 | `DeveloperPanel.vue` | Feed URL、版本覆盖、缓存目录、恢复默认 |

## 6. 状态与规则

| 状态对象 | 状态值 | 用户可见表现 | 可执行操作 | 不可执行操作 | 备注 |
|---|---|---|---|---|---|
| 版本状态 | 已是最新 | 无更新提示 | 正常使用 | 升级 | 检查通过 |
| 版本状态 | 弱更新可用 | 顶部提示、关于红点 | 立即升级、关闭提示、继续使用 | 无 | 用户可延后 |
| 版本状态 | 强更新可用 | 全局强更新弹窗 | 立即升级 | 关闭弹窗、继续使用 | 强阻断 |
| 下载状态 | 检查中 | 待确认用户提示 | 等待 | 重复检查 | 主进程记录日志 |
| 下载状态 | 下载中 | 待确认用户提示 | 等待 | 重复安装 | 具体进度展示需确认 |
| 下载状态 | 下载完成 | 升级入口可用 | 安装升级 | 无 | 主进程通知渲染层 |
| 下载状态 | 下载失败 | 用户侧提示待确认，日志记录 | 重试、打开下载地址 | 安装 | 异常主要进入日志 |
| Feed 状态 | 未配置 | 更新检查可能跳过 | 配置 Feed | 检查更新 | 开发调试场景 |
| Feed 状态 | 非法 | 返回无效地址提示 | 修改地址 | 检查更新 | 开发者面板 |

| 权限/条件 | 影响的功能 | 用户表现 | 提示文案 | 备注 |

## 9. 模块联动关系

- 更新机制与登录后初始化和主框架强联动，进入受保护页面和运行中都会检查版本。
- 关于弹窗由用户菜单触发，并复用版本检查结果显示红点与更新说明。
- 强更新弹窗由全局 App 承载，影响客户端所有模块可用性。
- 更新异常、下载完成、安装行为与本地日志模块联动。
- 开发者更新调试与开发者面板、日志查看器联动。

## 12. 面向重构的产品建议

| 建议 | 说明 |
|---|---|
| 明确强弱更新体验 | 强更新阻断、弱更新提醒应有统一视觉和文案 |
| 增加下载进度可见性 | 下载中和失败状态建议给用户明确反馈 |
| 保留关于入口升级能力 | 用户主动检查和理解版本信息依赖关于弹窗 |
| 统一异常去向 | 更新失败应给出重试、打开下载页、打开日志三类去向 |
| 开发者能力隔离 | Feed 和版本覆盖保留内部入口，避免普通用户接触 |

## 本模块产品还原结论

- 已确认功能：初始化检查、定时检查、弱更新顶部提示、关于红点、关于弹窗升级、强更新弹窗、下载完成安装、开发者更新调试。
- 新增发现功能：开发者 Feed URL、版本覆盖、更新缓存目录、强更新关闭行为限制。
- 待确认功能：更新下载中进度用户侧表现、更新失败用户侧提示、下载地址兜底在不同平台的展示方式。
- 疑似废弃功能：无明确疑似废弃功能。
- 复杂度最高的功能：强更新阻断与自动下载完成后的安装链路。
- 最适合 AI 接管或辅助的功能：更新失败解释、日志收集引导、版本变更说明摘要。
- 重构时最需要保留的能力：强更新阻断、弱更新提醒、关于升级、下载完成安装。
- 重构时可以弱化或合并的能力：开发者更新调试能力可归入统一开发者面板。
