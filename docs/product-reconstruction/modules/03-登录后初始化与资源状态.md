## 1. 模块定位

登录后初始化与资源状态模块负责把“用户已经登录”转换为“用户可以使用哪些客户端能力”。它在登录成功后加载用户身份、角色、全局配置、区域/集群、服务可用性、账户状态、可申请资源、跨区域路径、目标区域和若干运营初始化结果。

该模块解决四类产品问题：

- 用户进入客户端后看到哪些入口、哪些模块可用。
- 用户是否拥有计算资源、资源是否支持当前应用或功能。
- 区域是否维护、停用，账户是否欠费、停用、即将到期或预算超额。
- 当资源缺失或异常时，客户端如何引导用户购买、申请、刷新、联系客服或联系组管理员。

该模块是应用、作业、图形、文件、Shell、SClaw、费用中心、我的资源和全局任务能力的前置层。

## 3. 用户入口

| 入口名称 | 入口位置 | 触发方式 | 进入后页面/弹窗 | 备注 |
|---|---|---|---|---|
| 登录后初始化入口 | 登录成功后进入任意业务路由 | 自动触发 | 主框架或目标业务页 | 所有业务能力的前置流程 |
| 用户信息刷新入口 | 登录后路由切换完成 | 自动定时触发 | 用户状态更新 | 3 小时一次，最长 15 天 |
| 无资源缺省入口 | 文件页、当前作业页 | 无可用区域时自动展示 | 无资源缺省页 | 主框架内展示 |
| 开通计算资源弹窗入口 | 应用中心、应用提交、图形提交等 | 资源缺失或资源不支持应用时触发 | “开通计算资源”弹窗 | 支持按应用过滤可申请区域 |
| 账户告警入口 | 主框架顶部 | 登录后自动展示 | 账户告警横幅 | 可关闭，按用户记录 |
| 区域维护/停用入口 | 区域选择器、作业详情、应用详情等 | 区域状态异常时展示 | 标签、空状态或悬浮提示 | 维护中通常禁用操作 |
| Shell 区域入口 | 命令行窗口 | 选择区域或设置默认区域 | 区域卡片列表 | 显示账号、维护、停用、软件状态 |
| Shell 服务异常入口 | 命令行服务访问失败 | 自动展示 | 排查说明页 | 可联系客服 |
| 资源刷新入口 | 无资源弹窗 | 点击“刷新数据” | 刷新用户、区域、应用数据 | 用于用户刚完成资源开通后的恢复 |
| 订单/资源申请入口 | 无资源弹窗 | 点击查看订单、继续购买、购买资源、区域链接 | 外部控制台或商城页面 | 携带桌面端 Token 跳转 |

## 9. 模块联动关系

- 登录模块完成认证后进入本模块，本模块完成后主框架和业务模块获得可用状态。
- 客户端壳层根据区域列表、账户告警、无资源状态展示顶部告警、内联空状态和全局资源弹窗。
- 应用模块依赖目标区域、可安装区域、VM 支持区域、资源队列和账户状态，资源不支持时触发无资源弹窗。
- 作业管理依赖服务区域、作业管理服务、账户状态和区域维护状态；维护区域会影响作业详情内容展示。
- 文件模块依赖文件服务、文件传输服务、跨区域共享路径和默认文件区域；无资源时展示资源引导。
- Shell 模块依赖 Shell 服务、默认区域、账户状态、区域维护/停用和软件开通状态。
- 图形模块依赖图形服务、VM 支持区域、应用目标区域和账户状态。
- SClaw 入口依赖登录后白名单初始化；智能能力的授权码初始化也在本模块触发。
- 费用中心、我的资源、资源申请承接无资源引导、账户告警、预算超额和购买/充值路径。
- 帮助与客服模块承接区域维护、服务异常、无资源和测试区异常等断点。
- 数据埋点模块可围绕无资源、资源刷新、账户告警关闭和申请/购买入口建立事件字典。

## 12. 面向重构的产品建议

| 建议项 | 说明 |
|---|---|
| 建立统一资源状态字典 | 将区域状态、服务状态、账户状态、申请状态、订单状态统一命名和解释 |
| 抽象初始化状态机 | 登录后初始化涉及多路并发数据，建议定义用户、资源、配置、公告、版本、目标区域的完成和失败策略 |
| 拆分资源缺失与资源不匹配 | “没有任何资源”和“已有资源但不支持当前应用”是两种不同断点，应在产品文案和流程中分开 |
| 统一账户异常告警策略 | 停用、欠费、即将到期、预算超额都影响计算任务，应统一展示层级、关闭规则和后续动作 |
| 统一区域维护提示 | 区域选择器、按钮悬浮、空状态、作业详情中的维护提示应使用同一套文案 |
| 明确团队管理员与成员路径 | 管理员购买/查看订单，成员联系组管理员，代理用户按钮隐藏，需要形成权限矩阵 |
| 把资源刷新做成全局能力 | 资源刚开通后的“刷新数据”可作为全局状态刷新能力，供应用、文件、Shell、图形复用 |
| 降低初始化对业务首屏阻塞 | 公告、版本、优惠券、授权码等低感知能力可按产品优先级分层处理 |
| 资源状态适合由 AI 辅助解释 | AI 可以根据当前断点解释原因：缺资源、区域维护、账户停用、服务异常、应用不支持当前资源 |

## 本模块产品还原结论

- 已确认功能：登录后初始化编排、用户信息加载、角色识别、全局配置、公告和版本初始化、SClaw 白名单触发、区域列表、K8s 区域、云服务区域、区域维护/停用、服务能力状态、账户状态、账户告警、预算超额、无资源缺省页、开通计算资源弹窗、资源刷新、跨区域路径、应用目标区域、Shell 区域选择和服务异常提示。
- 新增发现功能：团队预算超额顶部提醒、账户告警可按用户关闭、全部区域账户停用时跳过活动类初始化、封装测试区说明弹窗、封装测试区服务异常空状态、Shell 区域卡片中的软件开通状态。
- 待确认功能：管理类/销售类角色的完整定义、资源申请 Failed 状态的产品含义、全部区域账户停用对各业务模块的阻断范围、活动优惠券和授权码初始化是否需要对用户可见。
- 疑似废弃功能：无明显废弃功能；`NoResource.vue` 中查看申请单入口存在注释块，需在资源申请模块继续确认。
- 复杂度最高的功能：资源状态与无资源引导，涉及用户角色、主站/单区、订单、申请、可申请区域、应用支持区域、资源刷新和外部跳转。
- 最适合 AI 接管或辅助的功能：解释无资源原因、推荐下一步资源申请路径、解释账户停用/欠费/到期影响、解释区域维护和服务异常、指导用户完成刷新恢复。
- 重构时最需要保留的能力：登录后初始化编排、区域/服务/账户三层状态、无资源引导、账户告警、资源刷新、区域维护/停用提示、跨模块共享的默认区域选择。
- 重构时可以弱化或合并的能力：订单/申请/购买入口可统一为资源服务入口；多处维护/停用 tooltip 可统一组件；封装测试区说明可归入 Shell 或应用打包的专项引导。
