## 1. 模块定位

登录与账号体系负责用户进入客户端前后的身份确认、会话维持、账号安全提醒、找回密码、重置密码、注册跳转和退出登录。

该模块在产品上承担四类能力：

- 允许用户通过账号密码或手机号验证码登录客户端。
- 帮助用户在登录前完成注册跳转、找回密码和重置密码。
- 在登录后维持会话，并在会话失效时统一回到登录页。
- 在退出登录时清理客户端状态，并保护运行中的传输任务和独立窗口。

该模块和登录后初始化、客户端壳层、个人中心、安全设置、SClaw、文件任务中心存在强关系。

## 9. 模块联动关系

- 登录成功后进入主框架，并触发用户信息、SClaw 白名单、资源、公告、版本、集群列表等初始化流程。
- 登录态影响所有业务模块，主框架、应用、作业、文件、图形、Shell、SClaw 均依赖有效登录态。
- 七日自动登录依赖本地 Token 和登录时间，自动登录成功后走同一条登录后初始化路径。
- 登录过期由业务请求统一触发，客户端会提示用户、回登录页、关闭二级窗口、清理缓存。
- 退出登录会调用主窗口任务检查，存在上传、下载、传输、Shell、图形等运行中窗口时提示用户确认。
- 退出登录会关闭 SClaw 后端服务并清理 SClaw 白名单状态，影响智能助手入口。
- 弱密码提醒会跳转外部个人资料安全页，和个人中心与账号安全模块联动。
- 注册、实名认证、费用中心、我的资源等账号相关外部入口通过桌面端 Token 跳转外部控制台。
- 登录页和登录结果会触发数据埋点，和数据埋点模块联动。

| 建立统一登录状态机 | 将未登录、登录中、成功、失败、超时、过期、退出中统一定义，减少页面和主进程状态分散 |
| 明确自动登录产品规则 | 七日自动登录、15 天用户信息刷新、Token 失效提示需要形成统一说明 |
| 统一验证码能力 | 账号登录、手机号登录、邮箱找回、手机号找回均涉及验证码，应统一文案、倒计时、频控和刷新规则 |
| 统一找回密码结果页 | 邮箱发送成功和重置成功均有 4 秒回登录，可抽象为一致的成功反馈体验 |
| 将弱密码提醒纳入账号安全域 | 弱密码弹窗和个人中心安全等级联动紧密，后续可在个人中心模块统一治理 |
| 保留退出前任务保护 | 退出登录影响运行中任务和独立窗口，是关键数据保护能力 |
| 明确外部账号系统边界 | 注册、实名、安全设置、费用、资源均跳转外部控制台，需要统一桌面端 Token 跳转规则 |
| 补充登录前合规入口确认 | 用户协议、隐私政策方法存在，登录页入口需要产品确认 |

## 本模块产品还原结论

- 已确认功能：账号密码登录、手机号验证码登录、登录方式切换、图片验证码、短信验证码、记住密码、七日自动登录、注册跳转、邮箱找回密码、手机号找回密码、重置密码、登录过期、退出登录、弱密码提醒、登录行为埋点。
- 新增发现功能：登录请求 65 秒超时保护、登录失败错误码精确文案、七日自动登录与记住密码联动、登录后 3 小时用户信息刷新且 15 天后停止、退出登录前运行任务保护、弱密码弹窗的本地关闭记录。
- 待确认功能：手机号登录历史用户标记的实际表现、手机号找回密码成功后的错误提示状态是否可见、登录前用户协议/隐私政策入口、SSO 第三方登录是否仍面向客户端用户。
- 疑似废弃功能：登录页注释中的引导组件、找回密码页底部注册提示注释块需要确认保留价值。
- 复杂度最高的功能：退出登录与登录过期处理，涉及任务保护、二级窗口关闭、SClaw 后端关闭、缓存清理和全局路由回登录。
- 最适合 AI 接管或辅助的功能：登录失败原因解释、找回密码路径引导、弱密码规则解释、登录过期后的恢复步骤、退出登录风险说明。
- 重构时最需要保留的能力：两种登录方式、自动登录、记住密码、找回/重置密码、登录过期统一处理、退出前任务保护、弱密码提醒。
- 重构时可以弱化或合并的能力：邮箱找回和手机号找回的成功反馈可合并；验证码错误、频控、服务异常文案可统一；注册和外部账号安全跳转可统一为账号服务出口。
