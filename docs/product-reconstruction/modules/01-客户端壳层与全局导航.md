## 1. 模块定位

客户端壳层与全局导航是 Electron 客户端登录后的产品承载层。它负责主窗口、独立窗口、全局导航、顶部工具区、用户菜单、全局弹窗、任务入口、窗口关闭保护、系统托盘和隐藏诊断入口。

该模块解决三个核心问题：

- 用户如何进入各个一级业务模块。
- 全局状态、任务、更新、公告、客服和账号菜单如何被用户发现。
- 多窗口、多任务、登录态失效和关闭行为如何保持可控。

主要用户场景包括：登录后进入应用/作业/文件/图形/SClaw；从顶部入口查看传输任务、更新、帮助、客服；从用户菜单进入个人中心、资源、费用和退出登录；关闭窗口时保护运行中的 Shell、上传、下载和传输任务。

## 3. 用户入口

| 入口名称 | 入口位置 | 触发方式 | 进入后页面/弹窗 | 备注 |
|---|---|---|---|---|

| 文件入口 | 左侧导航 | 点击文件图标 | 文件管理页 | 资源缺失时触发无资源引导 |

## 9. 模块联动关系

- 客户端壳层调用登录后初始化结果，决定 SClaw、命令行、任务类型、费用中心、用户菜单等入口是否显示。
- 左侧导航把用户带入应用、作业、图形、文件、Shell、SClaw，是所有主功能的一级入口。
- 顶部任务入口与文件模块、Shell 文件侧栏、下载路径弹窗、上传/下载/跨域/压缩任务状态联动。
- 顶部更新入口与更新机制、关于弹窗、强制更新弹窗联动。
- 顶部服务入口与在线客服窗口、日志系统、专属业务顾问外链联动。
- 用户菜单与个人中心、实名、费用中心、我的资源、关于、退出登录联动。
- 无资源引导与应用、文件、作业、图形、Shell 等资源依赖模块联动。
- 登录过期和退出登录会影响所有业务模块，并触发窗口清理、状态重置、SClaw 服务关闭。
- 窗口关闭保护与 Shell 连接、上传任务、下载任务、传输任务联动。
- 隐藏开发者面板与日志、更新、调试配置联动，面向内部支持和排障。

## 12. 面向重构的产品建议

| 建议项 | 说明 |
|---|---|
| 抽象统一的全局入口模型 | 左侧导航、顶部工具区、用户菜单、系统托盘、右键菜单应形成可配置入口字典，便于权限和运营商差异统一管理 |
| 抽象统一的窗口生命周期模型 | 主窗口、Shell、Transfer、上传、下载、VNC、VM、客服窗口的打开、聚焦、隐藏、关闭确认应有一致产品规则 |
| 保留任务保护能力 | 上传、下载、传输、Shell 关闭保护直接影响用户数据和连接安全，重构时需要优先保留 |
| 保留资源缺失引导能力 | 无资源弹窗是资源依赖功能的重要断点承接，应与费用、资源申请和刷新状态形成明确流程 |
| 合并支持服务入口 | 帮助、反馈、客服、日志可以在产品上统一为“帮助与支持”域，降低顶部工具区认知负担 |

| 明确隐藏功能治理 | 开发者面板、DeepSeek 路由、免责声明、独立上传/下载窗口需要确认面向用户、内部支持或历史兼容 |
| 明确平台差异规范 | macOS 和 Windows 的标题栏、菜单、关闭策略、用户菜单位置需要形成产品规范，避免重构后体验漂移 |
| 建立全局异常提示字典 | 登录过期、资源缺失、工具缺失、路径缺失、任务关闭风险应统一提示级别和用户动作 |

## 本模块产品还原结论

- 已确认功能：主窗口、登录后主框架、左侧导航、顶部工具区、任务入口、用户菜单、关于/更新、公告、新手引导、弱密码提醒、无资源引导、下载路径选择、工具下载、关闭保护、系统托盘、系统菜单、日志和客服入口。
- 新增发现功能：右键开发者面板、日志查看器、系统托盘、下载路径设为默认、FastTrans/Citrix 工具下载、关闭窗口任务保护、DeepSeek 路由、免责声明组件。
- 待确认功能：DeepSeek 页面正式入口、免责声明触发链路、SClaw Agent/Models 是否对最终用户开放、上传/下载独立窗口与 Transfer 聚合窗口的产品边界。
- 疑似废弃功能：免责声明弹窗显示链路、部分独立上传/下载窗口入口需要后续在任务中心模块确认。
- 复杂度最高的功能：顶部任务中心入口与多窗口关闭保护，涉及文件任务、下载路径、独立窗口、任务状态和关闭确认。
- 最适合 AI 接管或辅助的功能：帮助与支持入口、日志诊断解释、资源缺失后的路径建议、关闭任务风险说明、更新说明摘要。
- 重构时最需要保留的能力：左侧一级导航、全局任务角标、资源缺失引导、登录过期回登录、任务关闭保护、用户菜单退出清理、SClaw 白名单入口控制。
- 重构时可以弱化或合并的能力：帮助/反馈/客服/日志可合并为支持服务域，关于和更新提示可合并为版本中心，隐藏诊断入口可归入内部支持模式统一管理。
