## 1. 模块定位

命令行 / Shell 模块提供浏览器和 Electron 客户端内的远程集群终端能力，是用户直接操作计算环境、运行命令、管理文件、查看队列和调试应用环境的核心工具。它既可以从主导航进入，也可以从应用中心携带软件上下文进入，还可以在 AI/容器场景中以内嵌 WebShell 的形式出现。

该模块主要面向熟悉命令行的计算用户、管理员和需要排查作业/环境问题的用户。核心价值是把区域选择、节点连接、终端交互、多标签分屏、文件侧栏、上传下载和异常诊断整合到同一个工作区。

## 6. 状态与规则

| 状态对象 | 状态值 | 用户可见表现 | 可执行操作 | 不可执行操作 | 备注 |
|---|---|---|---|---|---|
| Shell 菜单 | 可见 | 左侧导航显示命令行 | 进入 Shell | 无 | 普通场景 |
| Shell 菜单 | BGP 隐藏 | 左侧导航不显示命令行 | 通过其他允许入口待确认 | 主导航进入 | BGP 运营商场景 |
| 区域 | 可用 | 区域卡片可选择 | 进入终端 | 无 | 有服务和账号 |
| 区域 | 维护/停用 | 区域卡片禁用，显示维护/停用图标 | 查看提示 | 进入 Shell | 受区域状态影响 |
| 账户 | 正常 | 可进入 Shell | 连接终端 | 无 | 普通用户路径 |
| 账户 | Disable/Trial | 账号停用/试用过期提示 | 管理员可按规则进入 | 普通用户进入 | 受角色影响 |
| 服务配置 | 正常 | 进入终端 | 使用 Shell | 无 | 直连或中转 |
| 服务配置 | 空配置/关闭 | 区域禁用或异常页 | 查看排查/客服 | 进入终端 | `service.eshell.empty/disable` |
| 连接 | connecting | 终端连接中 | 等待 | 输入命令 | 标签状态 |
| 连接 | connected | 可输入输出 | 命令、上传下载、分屏标签 | 无 | 正常工作态 |
| 连接 | disconnect | 断开提示 | 重连、排查 | 命令执行 | 异常态 |
| 连接 | autoReconnect | 自动重连提示 | 等待或手动重连 | 稳定执行命令 | 网络波动 |
| 模式 | direct | 支持多标签和更多会话能力 | 新建标签、分屏 | 无 | 直连 E-Shell |
| 模式 | forward | 中转代理 | 单 Pane 操作 | 多 Pane 分屏 | 覆盖范围待确认 |
| 标签 | 达到上限 | 新增被拦截 | 关闭旧标签 | 继续新增 | `MAX_TABS_COUNT` |

## 9. 模块联动关系

- Shell 与应用模块联动：应用卡片可携带软件版本、启动路径、环境变量、算例路径进入 Shell，并支持粘贴到终端。
- Shell 与文件模块联动：左侧嵌入文件管理，拖入路径执行 cd，拖入本地文件触发上传，上传完成刷新文件树。
- Shell 与全局任务中心联动：Shell 上传下载会打开或更新传输窗口。
- Shell 与作业模块联动：顶栏展示区域队列，调度命令帮助提供 Slurm/sc-client 命令，容器/训练作业可进入 WebShell。
- Shell 与资源/区域模块联动：区域服务地址、维护/停用、账号状态和服务健康决定入口可用性。
- Shell 与帮助/客服联动：服务异常、应用开通、排查失败提供帮助文档或客服入口。
- Shell 与本地客户端联动：Electron 独立窗口、关闭确认、文件选择、传输窗口均依赖客户端壳层能力。

## 12. 面向重构的产品建议

| 建议 | 说明 |
|---|---|
| 将入口、区域选择、终端工作区拆成清晰阶段 | Shell 体验依赖先选区域、再连接、再操作，阶段化有助于异常恢复 |
| 建立统一连接状态模型 | connecting、disconnect、autoReconnect、登录失败、服务异常需要统一文案和操作 |
| 保留多标签分屏能力 | 这是高频效率功能，需明确 direct 与 forward 模式的能力差异 |
| 强化文件侧栏和终端联动 | cd、上传、下载、刷新文件树是 Shell 与文件模块的核心协作能力 |
| 将疑难排查产品化 | 存储、目录、RSA、日志和客服入口适合作为连接失败的标准恢复流程 |
| 明确 BGP、管理员、维护角色差异 | 命令行是高权限入口，重构时需要完整权限矩阵 |
| 将应用上下文做成可复用信息面板 | 软件版本、环境变量、启动路径和算例路径对用户执行命令非常关键 |

## 本模块产品还原结论

- 已确认功能：主导航命令行、应用中心命令行、Electron 独立窗口、关闭确认、区域选择、区域切换、直连/中转模式、WebSocket 登录、重连、多标签、分屏、分屏方向、登录节点选择、文件侧栏、拖拽路径、ZMODEM 上传下载、全局传输窗口、队列信息、调度命令帮助、软件信息、容器 WebShell、疑难排查。
- 新增发现功能：应用开通提示、容器 WebShell、服务异常排查、错误日志导出、客服入口、拖拽路径 cd 与本地文件 rz 上传。
- 待确认功能：WebShell 路由注册来源、forward 模式覆盖范围、登录节点监控开关、Shell 客户端/密钥能力、BGP 深层路由直达策略。
- 疑似废弃功能：暂无明确废弃功能；节点 CPU/内存监控 UI 开关来源不明，归入待确认。
- 复杂度最高的功能：WebSocket 登录与重连、多标签分屏、ZMODEM 上传下载、疑难排查。
- 最适合 AI 接管或辅助的功能：连接失败排查、Shell 命令生成、Slurm 命令解释、软件环境变量配置、文件路径定位。
- 重构时最需要保留的能力：区域选择、终端连接、多标签分屏、文件联动、传输窗口联动、应用上下文。
- 重构时可以弱化或合并的能力：分散的错误提示、隐藏开关不清晰的节点监控、重复的区域切换确认入口。
