## 1. 模块定位

帮助、反馈、客服、日志模块为客户端提供问题自助解决、人工支持、产品反馈和故障排查入口。普通用户主要使用帮助文档、建议反馈、在线咨询和业务顾问联系方式；排障场景下用户可打开本地日志目录，开发者可进入日志查看器查看应用、SClaw、Gateway 日志。

## 9. 模块联动关系

- 帮助入口与应用模块、Shell 模块联动，提供场景化文档。
- 建议反馈通过登录 token 跳转控制台，连接外部反馈系统。
- 在线咨询与客服窗口由客户端壳层承载，遇到问题时可配合打开日志。
- 日志覆盖主应用、SClaw、Gateway、更新等能力，是排障和客服支持的基础材料。
- 开发者日志查看器与开发者菜单联动，面向内部调试和升级链路排查。

## 12. 面向重构的产品建议

| 建议 | 说明 |
|---|---|
| 保留问题闭环入口 | 帮助、反馈、客服、日志应作为右上角稳定入口 |
| 建立场景化帮助规则 | 应用、Shell、文件、图形等模块应提供上下文文档入口 |
| 顾问号码补齐空态 | 专属业务顾问缺失时需要清晰替代入口 |
| 日志入口标准化 | 普通用户打开目录，内部人员查看日志，两类路径应分层呈现 |
| Shell 工作流入口确认 | 代码保留的工作流能力需要明确上线计划 |

## 本模块产品还原结论

- 已确认功能：帮助文档、应用专属文档、Shell 帮助、建议反馈、在线咨询、专属业务顾问展示、打开日志、开发者日志查看器。
- 新增发现功能：客服窗口单实例聚焦、日志按 app/sclaw/gateway 分类、日志保留与轮转规则、Shell 工作流入口开关。
- 待确认功能：顾问号码为空时文案与点击行为、Shell 工作流入口后续形态、日志打开失败用户侧提示。
- 疑似废弃功能：Shell 工作流入口当前关闭，需要确认保留状态。
- 复杂度最高的功能：客服和日志形成的问题排查闭环。
- 最适合 AI 接管或辅助的功能：根据用户问题推荐帮助文档、引导打开日志、解释常见错误并生成反馈内容。
- 重构时最需要保留的能力：帮助文档、建议反馈、在线咨询、打开日志。
- 重构时可以弱化或合并的能力：开发者日志查看器可保留在内部入口，Shell 工作流入口可按产品决策合并。
